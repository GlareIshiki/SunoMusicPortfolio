# Grand Harmonic Archive

**A mystical music portfolio inspired by Fate/Grand Order's luxurious aesthetic**

古代の音楽図書館に保管された700以上の神秘的な音源アーカイブ。FGO(Fate/Grand Order)の重厚で豪華絢爛なデザインと、オーロラ・ドリームスケープの幻想的な美しさ、TACTICAL AUDIO ARCHIVEの硬派さを融合した、Suno AI音楽ポートフォリオサイトです。

## デザインコンセプト

**Grand Harmonic Archive**は、ゲーム公式サウンドトラックページのような豪華さと、映画的・シネマティックな美しさを追求したデザインです。

### デザインの特徴

本サイトは、FGOの召喚画面やストーリー選択画面のような豪華絢爛な演出を音楽ポートフォリオに適用しています。古代の音楽図書館という世界観のもと、神秘的で重厚な体験を提供します。

**豪華絢爛な配色**では、ゴールド(`#D4AF37`)をメインアクセントに、ディープパープル(`#6B21A8`)で神秘性を、クリムゾンレッド(`#DC2626`)で情熱を、ミッドナイトブルー(`#1E1B4B`)で深い背景を、シルバー(`#C0C0C0`)でセカンダリアクセントを表現しています。

**装飾的なUI要素**として、バロック様式の装飾フレーム、紋章・エンブレム、幾何学パターン(円形、六角形)、グラデーションオーバーレイ、光のエフェクト(グロウ、シャイン)を採用しています。

**エレガントなタイポグラフィ**では、Cinzel(セリフ体、古典的で威厳のある見出し)、Cormorant Garamond(エレガントなディスプレイフォント)、Inter(可読性の高い本文)、JetBrains Mono(データ表示)を組み合わせています。

**優雅なアニメーション**は、ゆったりとした動き(0.6-1.0s)、フェードイン・スライドイン、パーティクルエフェクト、グロウ・シャインアニメーションにより、FGOのような演出を実現しています。

**神秘的な演出**として、回転するエンブレム、mystical particles背景、グラデーションボーダー、装飾的なコーナーアクセントを配置しています。

## 主要機能

### 音楽再生

Spotifyライクなボトム固定プレイヤーを実装し、再生/停止、シーク、音量調整、次/前の曲、シャッフル、リピート(全曲/1曲/オフ)、リアルタイム進行状況表示に対応しています。プレイヤーは豪華な装飾フレームとグラデーション背景で、FGOのようなラグジュアリーな体験を提供します。

### 楽曲管理

720曲のダミーデータを用意し、検索機能(曲名、アーティスト、タグ)、ジャンルフィルター、並び替え(新しい順、古い順、タイトル順、アーティスト順、再生時間順)、グリッド/リスト表示切替を提供しています。楽曲カードは装飾的なフレームとグラデーションボーダーで豪華に演出されています。

### プレイリスト

キュレーションされたコレクションとして、プレイリスト一覧、プレイリスト詳細ページ、全曲再生機能を実装しています。各プレイリストは神秘的なビジュアルで表現されています。

### 楽曲詳細

カバーアート表示、生成プロンプト履歴(Generation Protocol)、タグ分類(Classification Tags)、メタデータ(ジャンル、再生時間、作成日)、カバー曲の場合は元曲リンク(Original Source)を表示します。詳細ページは装飾的なカードレイアウトで情報を整理しています。

### テーマ切替

ダークモード/ライトモード対応により、豪華な配色を維持したまま切替可能です。どちらのモードでもゴールドとパープルのアクセントカラーが際立ちます。

## 技術スタック

- **フレームワーク**: React 19 + TypeScript
- **ルーティング**: Wouter
- **スタイリング**: Tailwind CSS 4
- **アニメーション**: Framer Motion

## データ構造

各楽曲は以下のような構造で管理されています:

```typescript
interface Song {
  id: string;
  title: string;
  artist: string;
  genre: string;
  tags: string[];
  audioUrl: string;
  coverUrl: string;
  prompt: string;
  isCover: boolean;
  originalUrl?: string;
  createdAt: string;
  duration: number;
}
```

## 実際のデータへの置き換え

現在はダミーデータで動作していますが、実際のSuno AI楽曲データに置き換えるには:

1. `client/src/lib/mockData.ts`を編集
2. `mockSongs`配列を実際の楽曲データに置き換え
3. 音源URL(`audioUrl`)とカバーアート(`coverUrl`)を実際のURLに設定
4. プロンプト履歴やタグ情報を追加

## 今後の拡張案

### バックエンド統合

データベース(PostgreSQL/MongoDB)とファイルストレージ(S3)を追加することで、楽曲のアップロード・管理機能を実装できます。

画像の遅延読み込み、仮想スクロール、Service Worker(PWA化)、CDN配信により、さらなるパフォーマンス向上が期待できます。
